# LocalActions example (sdk-go)

Demo of `tnclient.NewLocalClient` — a standalone constructor that talks to
kwild's admin port for tn_local stream operations. No private key needed: the
admin server handles its own transport auth, and tn_local derives the
`data_provider` server-side from the node's secp256k1 key.

## Setup

1. Build and start the node with the **admin opt-in** task (default
   `single:start` keeps admin on a unix socket inside the container, which
   is unreachable from the host):

   ```bash
   cd /path/to/trufnetwork/node
   task docker:build:local      # first run, or after node code changes
   task single:start:admin      # exposes admin RPC on 127.0.0.1:8485
   ```

   `single:start:admin` layers `compose.admin.yaml` on top of the default
   compose file. It binds host port `8485` to **`127.0.0.1` only** and
   passes `--admin.listen 0.0.0.0:8485 --admin.notls` inside the container.
   Same trust model as the default unix socket, just over TCP — fine for
   production as long as the SDK runs on the same host as the node.

   For multi-host setups (SDK and node on different machines), set up
   mTLS or `--admin.pass` instead of plain HTTP, and override
   `TN_ADMIN_BIND=0.0.0.0`. See `node/docs/development.md` and
   `node/compose.admin.yaml` for the full options.

## Run — unsigned (works on `require_signature=false` nodes)

```bash
cd /path/to/trufnetwork/sdk-go
go run ./examples/local_actions_example/main.go
```

## Run — signed (required when the node has `require_signature=true`)

Start the node with auth required, then pass the operator's secp256k1
private key to the example. The example also demonstrates wrong-key
rejection before doing real work, so you can see the property close the
threat model end-to-end.

```bash
cd /path/to/trufnetwork/node
TN_LOCAL_REQUIRE_SIGNATURE=true task single:start:admin

# Pull the autogenerated operator key out of the running container
OPERATOR_KEY=$(docker exec tn-db cat /root/.kwild/nodekey.json | python3 -c "import sys,json;print(json.load(sys.stdin)['key'])")

cd /path/to/trufnetwork/sdk-go
go run ./examples/local_actions_example/main.go -operator-key=$OPERATOR_KEY
```

You should see:

```text
signing requests as operator 0x86170b...
connected to admin server at http://127.0.0.1:8485

[wrong-key check] signing as 0x690f23... (NOT the operator)
  rejected (expected): local.create_stream: jsonrpc.Error:
    code = -32602, message = tn_local: unauthenticated,
    data = {"reason": "signer is not this node's operator"}

primitive stream: stXXXXXX...
  inserted 5 records
  ...
```

If you forget the `-operator-key` flag against a `require_signature=true`
node, every call returns `tn_local: unauthenticated, reason: missing _auth` —
that's the SDK telling you it sent no signature where one was required.

Expected output (ids will differ — they're derived from the example name):

```text
connected to admin server at http://localhost:8485

primitive stream: stXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
  inserted 5 records
  read back: 5 records
    event_time=1704067200 value=1.000000000000000000
    event_time=1704153600 value=2.000000000000000000
    event_time=1704240000 value=3.000000000000000000
  index[0]=100.000000000000000000, index[-1]=500.000000000000000000

composed stream: stYYYY... (children: stAAA..., stBBB...)
  composed value at 1717200000: 150.000000000000000000 (expect ~150)
  after disable_taxonomy: 0 records (expect 0)

list_streams: 4/4 created streams visible
  data_provider: 0x... (server-derived)

```

## What this demonstrates

- `tnclient.NewLocalClient(adminURL)` — standalone, no signer required.
- `CreateStream` / `InsertRecords` / `GetRecord` / `GetIndex` — primitive workflow
  with parallel-array `LocalInsertRecordsInput`.
- `InsertTaxonomy` + `DisableTaxonomy` — composed stream lifecycle without
  consensus.
- `ListStreams` — every row is owned by the node, `DataProvider` is the
  node's own lowercased address.
- `DeleteStream` — cleanup; safe to call on a missing stream (we ignore
  errors in the deferred cleanup).

For projects that already use the full `tnclient.Client` (gateway-backed,
needs a signer), use `tnclient.NewClient(ctx, endpoint, tnclient.WithAdmin(adminURL), ...)`
and call `client.LoadLocalActions()` to share one client object.
